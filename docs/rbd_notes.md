# 一般网络建模提示（简要）

1. **先定义成功事件**：完成 N 次完整搬运循环，且在每个关键阶段存在满足任务要求的功能通路。
2. **再定系统边界**：哪些元件属于源点到汇点的功能网络？哪些只是监测、保护或辅助单元？
3. **从 RBD 过渡到一般网络**：
   - 串联链可以看作网络中的必经路径
   - 并联冗余可以看作网络中的可替代支路
   - 共享节点、交叉连接和多路径汇合是一般网络与传统 RBD 的主要区别
4. **可修系统分析**：
   - 不可修模型关注 `R(t)`
   - 可修模型关注 `A` 或运行期间的可用性指标
5. 常见错误：
   - 没有先定义成功事件就直接套公式
   - 把 duty 直接乘到可靠度或可用度上
   - 把一般网络重新简化成单一串并联结构，导致模型信息丢失
