# 实验1：完整搬运循环任务可靠度评估（桥式起重机电驱+机械控制系统，RBD+任务剖面）

## 一、实验目的
1. 掌握不可修复任务（non-repairable mission）条件下的 **RBD（可靠性框图）建模**与系统可靠度计算。
2. 学会将工程任务分解为 **完整搬运循环**（Pick–Lift–Travel–Place–Return），并用 **任务剖面/占空比** 将“阶段性工作”的部件纳入可靠度模型。
3. 能用 Python 自动化计算并输出规范化实验报告（Markdown），并对结果给出工程解释（薄弱环节、改进方向）。

---

## 二、工程背景与任务定义（完整搬运循环）
某自动化桥式起重机承担“工位A → 工位B”的托盘搬运。一个 **完整搬运循环**包含 5 个阶段：

1) Pick（取货）：下降到位、夹具闭合确认  
2) Lift（起升）：起升到安全高度、上限位有效  
3) Travel loaded（载荷行走）：小车运行到B、位置反馈有效  
4) Place（放货）：下降到位、夹具打开确认  
5) Return empty（空载返回）：小车回到A、位置反馈有效  

设单循环时长：`t_cyc = 6 min = 0.1 h`  
班次完成循环次数：`N`（建议取 N=60，对应 T=6h）  
班次任务总时间：`T = N * t_cyc`

**任务成功事件：**在整个班次内完成 N 次循环；任一循环任一阶段发生关键链路失效即任务失败。

**建模假设：**

- 不可修复任务：任务期间不考虑维修恢复
- 失效独立、指数分布（常失效率）
- 对“仅在部分阶段工作”的部件，采用占空比折算：$\lambda_\text{eff}=\lambda\times \text{duty}$ `λ_eff = λ * duty`

## 四、可靠性结构（RBD 功能图）
```text
C1 → C2 → [C3 || C4] → C5 → C6 → C7
     → Hoist(C8→C9→C10→C11)
     → Trolley(C12→C13→C14)
     → [C15||C16] → [C17||C18] → C19
```

* `[C3 || C4]`：整流模块 **并联冗余 1oo2**
* `Hoist(...)`：起升链 **串联**
* `Trolley(...)`：小车链 **串联**
* `[C15||C16]`：上限位冗余 **1oo2**
* `[C17||C18]`：编码器冗余 **1oo2**
* 总体逻辑：关键链路按任务成功事件串联组合

### 任务 1：建模与公式（必须手工完成）

1. 明确任务时间 `T=N*t_cyc`，写出各元件 `λ_eff = λ*duty`。
2. 写出单元件可靠度：`R_i(T)=exp(-λ_eff*T)`。
3. 写出 1oo2 并联可靠度：`R=1-(1-R_A)(1-R_B)`。
4. 写出系统可靠度表达式 `R_sys(T)`（按本实验 RBD）。

## 十、提示（常见易错点）

* duty 只用于折算“有效失效率”：`λ_eff = λ*duty`，不要把 duty 直接乘到可靠度上。
* 1oo2 并联：`R=1-(1-R_A)(1-R_B)`，不是 `(R_A+R_B)/2`。
* 串联系统：可靠度相乘；对数空间更稳定（可选）。
* 单位一致：λ 用 1/h，T 用 h。
