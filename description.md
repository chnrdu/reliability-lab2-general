# 实验2：一般网络与可修系统可靠性分析（桥式起重机机械系统，lab2-general）

## 一、实验目的
1. 掌握在同一工程对象上，从 **RBD 串并联系统** 过渡到 **一般网络系统** 的可靠性建模方法。
2. 理解不可修系统可靠度与可修系统可用度的差异，掌握引入维修恢复参数后的基本分析流程。
3. 学会将桥式起重机机械系统抽象为一般功能网络，并基于任务剖面分析关键路径、薄弱环节和改进方向。
4. 使用 Python 自动化计算并输出规范化实验报告。

---

## 二、工程背景与任务定义
某自动化桥式起重机承担“工位A → 工位B”的托盘搬运。仍采用一个 **完整搬运循环** 作为任务背景，包含 5 个阶段：

1) Pick（取货）：下降到位、夹具闭合确认  
2) Lift（起升）：起升到安全高度、上限位有效  
3) Travel loaded（载荷行走）：小车运行到B、位置反馈有效  
4) Place（放货）：下降到位、夹具打开确认  
5) Return empty（空载返回）：小车回到A、位置反馈有效  

设单循环时长：`t_cyc = 6 min = 0.1 h`  
班次完成循环次数：`N`（建议取 N=60，对应 T=6h）  
班次任务总时间：`T = N * t_cyc`

**任务成功事件：**

- 对不可修系统模型：在整个班次内完成 N 次循环；任一关键功能链中断即任务失败。
- 对一般网络模型：在每次任务期间，系统源点到汇点之间存在满足成功条件的可用路径。
- 对可修系统模型：在给定运行期内，系统通过失效与维修交替运行，考察其可用度与功能保持能力。

**建模假设：**

- 元件失效相互独立，失效服从指数分布（常失效率）
- 对阶段性工作的部件，采用占空比折算：`λ_eff = λ * duty`
- 可修系统中维修过程近似服从指数分布，维修率记为 `μ`
- 在本实验中，先采用工程上可计算的简化模型，不要求建立完整全状态马尔可夫链

## 三、系统对象与建模抽象
仍采用原桥式起重机机械系统中的元件集合，包括供电、控制、起升、小车、限位、编码器和夹具等部件。

本实验要求在同一物理系统上完成两种抽象：

### 1) 一般网络抽象
- 将元件和功能连接关系表示为一般网络，而非仅限串联/并联嵌套
- 允许一个部件参与多条功能路径
- 以“源点到汇点连通”或“关键功能节点连通”作为系统成功判据

### 2) 可修系统抽象
- 对每个关键部件引入维修参数（如 `μ` 或 `MTTR`）
- 计算单部件稳态可用度
- 基于系统结构计算系统层可用度，并与不可修系统可靠度做对比

### 3) 占空比与任务剖面
- `duty=1` 表示全程工作；否则为阶段性动作占比
- 任务剖面继续用于计算 `λ_eff`
- 对可修系统，可根据实验要求选择是否继续使用 `λ_eff` 参与稳态指标计算，但必须在报告中说明口径

## 四、结构建模要求

### 1) 从 RBD 到一般网络
你需要将原有机械系统中的关键功能链，改写为一般网络结构。建议至少包含以下类型的节点或连接关系：

- 共用供电与控制链
- 起升执行链
- 小车执行链
- 位置/限位反馈链
- 夹具动作链
- 至少一组可替代路径、交叉连接或共享节点

一般网络模型不要求局限于 `series/parallel` 二叉结构，而应明确：

- 网络节点和连接关系
- 源点与汇点
- 系统成功判据

### 2) 可修系统参数
在一般网络模型基础上，为关键元件增加维修参数，至少完成：

- 单元件失效率 `λ`
- 单元件维修率 `μ` 或平均维修时间 `MTTR`
- 单元件稳态可用度 `A_i`
- 系统层可用度或等效运行能力指标

## 九、提示（常见易错点）

* duty 只用于折算“有效失效率”：`λ_eff = λ*duty`，不要把 duty 直接乘到可靠度上。
* 可修系统常用稳态可用度：`A = μ / (λ + μ)`，前提是失效与维修过程采用指数分布近似。
* 一般网络模型必须先明确定义成功事件，再讨论可靠度或可用度。
* 单位一致：λ 用 1/h，T 用 h。
